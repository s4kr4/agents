# エラーハンドリング

## 基本原則

1. **エラーは握りつぶさない** - 必ずログを記録するか、上位に伝播
2. **ユーザーフレンドリーなメッセージ** - 技術的詳細は隠す
3. **適切な粒度で捕捉** - 広すぎる catch は避ける
4. **リソースのクリーンアップ** - finally や using を活用

## エラーログの指針

### 含めるべき情報

- タイムスタンプ
- エラーメッセージ
- スタックトレース
- 関連するコンテキスト（ユーザーID、リクエストIDなど）
- エラーの重大度（ERROR, WARN, INFO）

### 含めてはいけない情報

- パスワード
- APIキー
- 個人情報（必要最小限に）
