# 作業中のエラー対応ガイドライン

## 基本原則

**作業中にエラーが発生した場合、イレギュラーな方法やコマンドで無理やり解決しようとしない。**

エラーを強引に回避しようとすることは、根本原因を隠蔽し、予期しない副作用を生む可能性がある。

## 対象となるエラーの例

- 権限エラー（Permission denied、sudo が必要な操作）
- ファイル操作の失敗（読み取り専用、ロック中、存在しないパスなど）
- コマンドの失敗（exit code != 0、依存ツール未インストールなど）
- ネットワークエラー（接続失敗、タイムアウトなど）

## 禁止する対応

- `sudo` や権限昇格を勝手に試みる
- `--force`、`--no-verify`、`--skip-checks` などの安全チェックを無効化するフラグを使用する
- エラーを握りつぶして処理を続行する
- 失敗したコマンドを変化なく繰り返しリトライする
- 問題のあるファイルを削除・上書きして解決しようとする
- 本来のフローを迂回する回避策を黙って実行する

## 正しい対応フロー

1. **与えられた権限内で解決できるか精査する**
   - 別のアプローチ・コマンド・ファイルパスで同じ結果を得られるか検討する
   - 権限昇格やフラグ無効化を伴わずに解決できる方法があれば、それを試みる
   - 解決策が見当たらない、またはリスクが不明な場合は次のステップへ進む
2. **エラー内容をユーザーに報告する**
   - 何が起きたか（エラーメッセージ、コマンド、対象ファイル）
   - なぜ発生したと考えられるか
   - ステップ1で検討した代替案とそれが使えなかった理由
3. **解決策の選択肢を提示する**
   - ユーザーが手動で対応できる手順を示す
   - 複数の選択肢がある場合はそれぞれのトレードオフを説明する
4. **ユーザーの判断を仰ぐ**
   - 自己判断で強行せず、次のアクションはユーザーに決めてもらう

## 報告の例

```
❌ 権限エラーが発生しました。

コマンド: chmod 755 /etc/app/config.json
エラー: Permission denied

このファイルはシステム領域にあるため、管理者権限が必要です。
以下のコマンドをターミナルで手動実行してください：

  sudo chmod 755 /etc/app/config.json

実行後、作業を再開できます。
```
